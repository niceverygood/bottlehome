# 타겟 기업 리서치 스크립트

`find-target-companies.mjs` — 업종·지역 키워드로 카카오 로컬 검색(장소 검색) API를 조회해
후보 업체 목록을 CSV로 뽑아준다.

## 이 스크립트가 하지 않는 것 (중요)

**이메일 주소를 수집하지 않고, 어떤 메일도 자동으로 보내지 않는다.**
정보통신망법 제50조의2는 이메일 주소를 자동으로 수집하는 프로그램 자체를 금지한다 — 수신 동의
여부와 무관하게, 그런 도구를 쓰는 것 자체가 불법이다. 그래서 이 스크립트는 공개된 **업체명·업종·
주소·대표 전화번호**만 모은다. 이것들은 개인 이메일이 아니라 사업장 정보이므로 위 조항과 무관하다.

## 사용 흐름

```
1. 이 스크립트로 후보 업체 리스트업 (자동)
        ↓
2. 담당자가 직접 전화·방문으로 연락 (사람이 함)
        ↓
3. 상대가 관심 있으면 이메일 주소를 직접 받음 (동의 확보)
        ↓
4. 그 이메일로 /erpanalysis/ 진단 링크 안내 또는 뉴스레터 구독(더블 옵트인) 권유
        ↓
5. 여기서부터는 erp-nurture / erp-subscribe 가 자동으로 이어받는다
```

## 준비물

카카오 디벨로퍼스에서 무료 REST API 키 발급 (즉시 발급, 카드 등록 불필요):
1. https://developers.kakao.com → 로그인 → **내 애플리케이션** → **애플리케이션 추가하기**
2. 앱 생성 후 **앱 키 → REST API 키** 복사

## 사용법

```bash
KAKAO_REST_API_KEY=발급받은키 node scripts/find-target-companies.mjs \
  --query "정밀부품 제조" --region "인천" --pages 3 --out leads.csv
```

| 옵션 | 설명 | 기본값 |
|---|---|---|
| `--query` | 업종·업태 키워드 (필수) | - |
| `--region` | 지역 키워드 (선택) | - |
| `--pages` | 페이지당 15건, 최대 3페이지 (카카오 API 자체 상한 45건) | 3 |
| `--out` | 저장할 CSV 경로 | `out/target-companies.csv` |

출력 CSV 컬럼: 업체명 · 업종 분류 · 전화번호 · 주소 · 카카오맵 링크 · 접촉 상태(빈 칸, 엑셀에서
직접 기록용) · 메모.

## 검색어 예시

ERP 진단 서비스의 타겟(제조·도소매·서비스 중소기업)에 맞춰 이렇게 조합해 보면 좋다:

- `"사출성형 제조" --region "안산"`
- `"식품 제조" --region "김포"`
- `"화장품 OEM" --region "인천"`
- `"물류센터" --region "용인"`

45건 상한 때문에 한 번에 넓은 키워드보다 좁은 업종+지역 조합을 여러 번 돌리는 편이 결과가 좋다.
